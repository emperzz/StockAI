# 🚀 StockAI 前端使用指南

## 📋 快速开始

### 1. 启动前端服务

```bash
# 激活环境
conda activate open_manus

# 启动前端
python start_frontend.py
```

### 2. 访问界面

打开浏览器访问：**http://localhost:7860**

## 🎯 功能说明

### 📊 主要功能

1. **股票数据查询**
   - 输入6位股票代码（如：000001）
   - 获取实时股票数据
   - 显示K线图

2. **数据分析**
   - 显示股票基本信息
   - 计算涨跌幅
   - 展示历史数据统计

3. **可视化图表**
   - 交互式K线图
   - 暗色主题适配
   - 响应式设计

### 🎮 使用方法

1. **输入股票代码**
   - 在输入框中输入6位股票代码
   - 例如：000001（平安银行）

2. **点击分析按钮**
   - 点击"🔍 分析股票"按钮
   - 系统将获取数据并进行分析

3. **查看结果**
   - 分析结果：显示股票基本信息和统计数据
   - 数据表格：显示历史交易数据
   - K线图：可视化价格走势

4. **快速测试**
   - 使用页面底部的快速测试按钮
   - 一键填入常用股票代码

### 📋 示例股票代码

| 代码 | 名称 | 类型 |
|------|------|------|
| 000001 | 平安银行 | 银行股 |
| 000002 | 万科A | 地产股 |
| 600000 | 浦发银行 | 银行股 |
| 600036 | 招商银行 | 银行股 |

## 🛠️ 技术架构

### 前端框架
- **Gradio**: 现代化的Web界面框架
- **Plotly**: 交互式图表库
- **响应式设计**: 适配不同屏幕尺寸

### 数据源
- **AKShare**: 中国股市数据API
- **实时数据**: 获取最新股票信息
- **历史数据**: 支持多时间周期查询

### 功能模块
```
gradio_app.py
├── 数据获取模块
│   ├── get_stock_info() - 获取股票基本信息
│   └── get_stock_data() - 获取历史数据
├── 可视化模块
│   └── create_stock_chart() - 创建K线图
├── 分析模块
│   └── analyze_stock() - 综合分析
└── 界面模块
    └── create_gradio_app() - 创建Web界面
```

## 🔧 故障排除

### 常见问题

1. **端口被占用**
   ```bash
   # 查看端口占用
   lsof -i :7860
   
   # 杀死占用进程
   kill -9 <PID>
   ```

2. **依赖缺失**
   ```bash
   # 重新安装依赖
   pip install -r my_agent/requirements.txt
   ```

3. **数据获取失败**
   - 检查网络连接
   - 确认股票代码正确
   - 查看控制台错误信息

### 日志查看

启动时会显示详细日志，包括：
- 服务启动状态
- 数据获取过程
- 错误信息
